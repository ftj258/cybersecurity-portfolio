# AWS IAM Best Practices

**Objective:** Document key governance practices to secure AWS identity access.

---

## Key Principles

• Least Privilege Access  
• Role-based IAM policies  
• MFA enforcement  
• Rotate access keys

---

## Recommendations

• Use IAM roles instead of long-term keys  
• Grant only required service permissions  
• Apply permission boundaries
• Enable MFA for privileged users

---

## Logging & Monitoring

• CloudTrail enabled across all regions  
• AWS Config rule compliance alerts  
• GuardDuty identity anomaly findings

---

## Skills Demonstrated

• AWS governance design  
• Identity access validation
